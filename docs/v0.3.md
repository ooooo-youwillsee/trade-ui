# v0.3

## Summary

`v0.3` 增强现货网格和合约网格的仓位控制与详情可读性。该版本新增单格仓位递增参数，让网格可以在总投资额固定的前提下按逆势方向分配仓位；同时优化网格详情页和编辑页的交互，让挂单价格、挂单仓位和成交状态更容易查看。

## Key Features

- 网格仓位递增：
  - 现货网格和合约网格都支持单格仓位递增。
  - 支持比例递增和 USDT 数量递增两种模式。
  - 默认递增值为 `0`，旧策略保持等额分配。
  - 做多时低价网格仓位更大，做空时高价网格仓位更大。
- 总投资额固定：
  - 递增只改变每个网格之间的仓位权重。
  - 现货按每格投入金额分配。
  - 合约按每格保证金分配，并按杠杆换算名义仓位。
- 网格详情挂单列表：
  - 详情页“网格价格”区域默认折叠。
  - 展开后以列表展示每个实际挂单格。
  - 每行展示挂单价格、挂单仓位和是否成交。
  - 成交状态沿用当前网格计算逻辑，和持仓、盈亏结果保持一致。
- 编辑页输入优化：
  - “仓位递增方式”和“单格递增”合并为一个输入控件。
  - 输入框右侧可切换比例或数量。
  - 选择比例时后缀显示 `%`。
  - 选择数量时后缀显示 `USDT`。

## Calculation Notes

- 网格价格仍由上下限价格、网格数量和网格模式生成。
- 仓位递增不会改变网格价格，只影响每个挂单格的仓位金额。
- 比例递增会按递增比例生成权重，再归一化到总投资额。
- 数量递增会按固定 USDT 差额分配每格仓位，并校验总投资额是否足够。
- 详情页挂单列表展示 `gridCount` 个实际挂单格，不展示额外边界价格点。

## UI Notes

- 现货网格编辑页和合约网格编辑页保持一致的仓位递增输入方式。
- 网格详情页保留原有参数信息、核心指标和当前持仓区域。
- 网格价格区域从标签云改为可折叠挂单列表，避免网格数量较多时页面过长。
- 合约挂单列表的仓位列显示保证金，现货挂单列表的仓位列显示投入金额。

## Assumptions

- 本版本仍只做本地策略计算和保存，不接入交易所 API。
- 单格数量模式以 USDT 金额差解释，不表示币数量或合约张数。
- 成交状态是策略计算状态，不代表真实交易所订单状态。
- 旧策略数据不迁移；缺失的新字段通过默认值保持兼容。
