# v1.1 变更记录

## 概览

v1.1 将交易策略从单一合约网格扩展为四条独立策略线：

- 合约网格 `contractGrid`
- 合约马丁 `contractMartingale`
- 现货网格 `spotGrid`
- 现货马丁 `spotMartingale`

底部导航调整为“合约、现货、股票、我的”。合约和现货作为一级市场入口，分别支持添加网格策略和马丁策略；股票入口暂时保持占位。

## 功能变化

- 新增独立马丁策略计算能力：
  - 支持合约和现货两种模式。
  - 支持做多/做空。
  - 支持首单金额、加仓倍数、最大层数、触发幅度、止盈比例、资金上限等参数。
  - 合约马丁额外支持杠杆、追加保证金、维持保证金率、估算强平价和强平距离。
  - 现货马丁隐藏合约参数和强平风险，只展示资金、均价、止盈和层级结果。
- 新增现货网格策略：
  - 支持上下限价格、入场价、当前价、等差/等比网格、网格数量、方向和投入金额。
  - 结果展示持仓均价、当前权益、浮动盈亏、持仓数量、单格收益率、区间收益率和网格价格。
- 合约网格保留原有计算口径：
  - 继续展示强平价、保证金、名义仓位、浮盈亏、网格收益率等指标。

## 导航与路由

- 底部导航改为：
  - `/contract` 合约
  - `/spot` 现货
  - `/stock` 股票
  - `/mine` 我的
- 合约页新增菜单支持：
  - `/contract/grid/new`
  - `/contract/martingale/new`
- 现货页新增菜单支持：
  - `/spot/grid/new`
  - `/spot/martingale/new`
- 详情和编辑路由保持语义化：
  - `/contract/grid/:id`
  - `/contract/grid/:id/edit`
  - `/contract/martingale/:id`
  - `/contract/martingale/:id/edit`
  - `/spot/grid/:id`
  - `/spot/grid/:id/edit`
  - `/spot/martingale/:id`
  - `/spot/martingale/:id/edit`

- 合约网格：
  - `ContractGridEditPage.vue`
  - `ContractGridDetailPage.vue`
  - `ContractGridForm.vue`
  - `ContractGridResults.vue`
- 合约马丁：
  - `ContractMartingaleEditPage.vue`
  - `ContractMartingaleDetailPage.vue`
  - `ContractMartingaleForm.vue`
  - `ContractMartingaleResults.vue`
- 现货网格：
  - `SpotGridEditPage.vue`
  - `SpotGridDetailPage.vue`
  - `SpotGridForm.vue`
  - `SpotGridResults.vue`
- 现货马丁：
  - `SpotMartingaleEditPage.vue`
  - `SpotMartingaleDetailPage.vue`
  - `SpotMartingaleForm.vue`
  - `SpotMartingaleResults.vue`

公共页面和组件命名也进行了规范化：

- `AppBottomNav.vue` 作为应用底部导航。
- `MineProfilePanel.vue` 作为“我的”页内容组件。

已删除旧的混合或过期入口：

- `ContractListView.vue`

- `MartingaleStrategyPage.vue`

## 状态与默认值

四类策略分别使用独立 store 和默认参数文件：

- `useContractGridStrategies.js`
- `useContractMartingaleStrategies.js`
- `useSpotGridStrategies.js`
- `useSpotMartingaleStrategies.js`
- `contractGridDefaults.js`
- `contractMartingaleDefaults.js`
- `spotGridDefaults.js`
- `spotMartingaleDefaults.js`

合约网格默认参数文件完成命名规范化：

- `strategyDefaults.js` 重命名为 `contractGridDefaults.js`
- `defaultInput` 重命名为 `defaultContractGridInput`
- `presets` 重命名为 `contractGridPresets`

马丁计算核心保留为纯计算模块：

- `martingale.js`

现货网格计算模块独立为：

- `spotGrid.js`

## 本地存储

四类策略使用独立 localStorage key，不迁移旧数据：

- 合约网格：`contract-grid-v2-strategies`
- 合约马丁：`contract-martingale-strategies`
- 现货网格：`spot-grid-strategies`
- 现货马丁：`spot-martingale-strategies`
