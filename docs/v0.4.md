# v0.4

## Summary

`v0.4` 为合约网格新增中性方向。中性合约网格在同一组价格区间内拆分多腿和空腿：入场价下方按做多网格计算，入场价上方按做空网格计算，并在列表、详情和挂单明细中保持可读展示。

## Key Features

- 中性合约网格：
  - 合约网格方向新增 `neutral`，与原有 `long`、`short` 共用同一策略入口和本地存储。
  - 当前价格低于入场价时触发下方多腿网格。
  - 当前价格高于入场价时触发上方空腿网格。
  - 创建时建仓会同时生成入场价两侧的多腿和空腿计划仓位。
- 双腿结果：
  - 计算结果新增 `longLeg` 和 `shortLeg`，分别包含成交网格、名义仓位、浮动盈亏、权益和强平价。
  - 合计字段继续保留 `currentNotional`、`floatingProfitLoss`、`currentEquity`、`liquidationPrice` 等兼容展示字段。
- 明细展示：
  - 合约网格详情页新增“双腿风险”区块，展示多腿/空腿强平价、名义仓位和浮动盈亏。
  - 网格挂单列表新增方向标签，标识每格属于做多腿或做空腿。
  - 合约策略列表中性卡片显示“中性”标签，不再使用上涨或下跌图标暗示单边方向。

## Calculation Notes

- 网格价格仍由上下限价格、网格数量和网格模式生成，不新增独立中性价格算法。
- 中性模式按入场价分界：低于或等于入场价的挂单按多腿处理，高于入场价的挂单按空腿处理。
- 兼容字段 `estimatedGridLiquidationPrice` 和 `liquidationPrice` 在中性模式下取距离当前价格最近的一侧强平价。
- 中性模式的单格收益率和区间收益率取多空两侧中更保守的一侧，避免列表指标过度乐观。
- 追加保证金按多腿和空腿已占用保证金比例分摊；未形成仓位时两边均分。

## UI Notes

- 合约网格编辑页方向选项新增“中性”。
- 详情页顶部方向徽章支持“中性”，并使用双向图标。
- 详情页仅在中性模式下展示“双腿风险”，原做多/做空详情保持原结构。
- 列表页继续复用合约网格卡片，不新增独立路由、入口或策略类型。

## Verification

- `npm.cmd test` 通过，覆盖 5 个测试文件、31 个测试。
- `npm.cmd run build` 通过，Vite production build 成功。
- 新增测试覆盖中性下跌触发多腿、上涨触发空腿、创建时双腿建仓、挂单方向、兼容强平价字段和健康度距离计算。

## Assumptions

- 本版本仍只做本地策略计算和保存，不接入交易所 API。
- 本版本只修改 `trade-ui`，不修改 `trade-server`。
- 中性合约网格作为合约网格的方向值实现，不新增 localStorage key。
- 旧策略缺少新字段时继续通过默认值兼容。
